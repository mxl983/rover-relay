Battery · avg minutes per 1% (by SOC band)
Mean time to lose 1% SOC while discharging (video on, MQTT session). Only heartbeat pairs ≥ ~45s apart are used (same rule as active drain rate) so rapid samples do not distort minutes-per-1%. Intervals while charging are excluded. Shorter bar = faster drain in that region — useful for spotting weak capacity bands.

Battery drain rate — active vs idle
Per local day: active (ended sessions) = Σ drop ÷ Σ minutes per calendar-day slice. Active (current MQTT session still open) = one running average from session start to the latest heartbeat, counted on today only — as time passes the denominator grows so the point falls if SOC levels off. Slices need ≥ ~45s. Idle = Σ drop ÷ Σ idle minutes (SOC before mqtt_power_off vs after power_on). Charging excluded.
